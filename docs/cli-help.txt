chosko-llm — manage global Claude Code commands and skills.

Usage:
  chosko-llm ls                       List all features with installed and latest versions.
  chosko-llm ls --installed           Show only features currently installed under $CLAUDE_HOME.
  chosko-llm ls --available           Show only features present in the managed clone.

  chosko-llm add <feature>            Copy a feature from the managed clone into $CLAUDE_HOME.
                                      Errors if the feature is already installed (use 'update').
  chosko-llm rm  <feature>            Delete an installed feature from $CLAUDE_HOME.
  chosko-llm update <feature>         Re-copy a feature from the managed clone (installs if missing).
  chosko-llm update --all             Update every installed feature for which a source exists.

  chosko-llm upgrade                  git pull the managed clone and refresh the CLI proxy.
                                      Does NOT re-install features — use 'update --all' for that.

  chosko-llm task-impl <N> [<N>…]     Drive aider+Ollama through the 8-step /task-implement
  chosko-llm task-impl all            sequence for one or more backlog tasks in the current
                                      project. Requires /task-setup to have been run first.
                                      One commit per task.
                                      Run 'chosko-llm task-impl --help' for full options.

  chosko-llm help                     Show this help.

Feature names:
  Bare names (e.g. 'refactor-reviewer') match commands and skills. If a name exists as both,
  disambiguate with 'command:<name>' or 'skill:<name>'.

Environment:
  CHOSKO_LLM_HOME   Path to the managed clone. Default: ~/.chosko-llm
  CLAUDE_HOME       Path to Claude Code's user directory. Default: ~/.claude
  BIN_DIR           Where the CLI proxy is installed. Default: ~/bin
  NO_COLOR          Set to disable colored log output.
