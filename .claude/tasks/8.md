# Task 8 — External LLM prompts: pin English, forbid stub-only outputs, mark non-interactive

## Description

A real run of `chosko-llm task-impl 9` against `ollama/qwen2.5-coder:14b`
exposed two prompt-quality bugs in the `.claude/external/*-prompt.md`
templates that `/task-setup` writes:

1. **Wrong language.** The model emitted Italian prose ("Per completare il
   task richiesto…", "Fate mi sapere se posso procedere…"). Neither
   `tests-prompt.md` nor `implement-prompt.md` says anything about output
   language, so the model picks one based on locale / repo cues / luck.
   This wastes tokens on translation effort and produces unreviewable
   commit-adjacent prose.

2. **Stub-only test file + chat-style sign-off.** The test pass produced
   `tests/test_report.py` containing two `@pytest.fixture` mocks and a
   single `def test_report_command(...): pass` body, then the model ended
   with "Fate mi sapere se posso procedere con l'aggiunta di ulteriore
   contenuto…" — a request for a follow-up turn. But aider was invoked
   with `--message` (one-shot, non-interactive); there is no follow-up.
   The orchestrator at `scripts/cmd-task-impl.sh:188-195` then runs the
   affected tests, sees them pass trivially (because the body is `pass`),
   and aborts with `new tests passed unexpectedly`.

   Root cause: the prompt never tells the model the invocation is
   single-shot, and "Output discipline" does not forbid scaffolding-only
   output (empty test bodies, `TODO` placeholders, requests for
   confirmation).

Both bugs live in the `EXTERNAL IMPLEMENT-PROMPT TEMPLATE` and `EXTERNAL
TESTS-PROMPT TEMPLATE` BEGIN/END blocks inside `commands/task-setup.md`
(lines ~373-421 and ~433-482). Editing those template blocks is the
canonical fix; freshly-initialized projects then get the corrected
prompts. Existing projects re-acquire them by deleting the affected file
and re-running `/task-setup`, which is already covered by smoke scenario
4 ("Re-running on a partially initialized project").

A defense-in-depth fix also lives in `run_aider` at
`scripts/cmd-task-impl.sh:119-132`: the per-invocation `--message`
strings (lines 184-185, 204-206, 240-241) get an `Respond in English.`
suffix so even an out-of-date prompt file can't slip Italian past us.

### Behavior change

After this task lands, both `EXTERNAL TESTS-PROMPT TEMPLATE` and
`EXTERNAL IMPLEMENT-PROMPT TEMPLATE` blocks in `commands/task-setup.md`
contain:

1. A new top-level section **`## Execution model`** placed immediately
   after `## Inputs`, with this content (parallel in both prompts; the
   examples differ by pass):

   - You are running in a **single-shot, non-interactive aider
     invocation** (driven by `--message`). There is no follow-up turn.
     You cannot ask the user to confirm, cannot request additional
     context, and cannot defer work to "the next message".
   - **Respond in English.** All prose, code comments, commit-adjacent
     text, and any explanation you emit must be in English regardless
     of the language used in the repo or the task body.
   - If a piece of information is missing, do not ask — encode the
     best-guess behavior and add a brief code comment noting the
     assumption. If the situation is genuinely a Stop condition (see
     below), stop and report; do not request a follow-up turn.

2. An additional bullet at the **end of `## Output discipline`**, in
   the **tests-prompt** only:

   - **No scaffolding-only output.** Every test named in the task
     body's `Tests` section must have real assertions in this single
     pass. No `pass`-bodied test functions, no `TODO` / `# implement
     here` placeholders, no fixture-only files. If a behaviour cannot
     be asserted yet (e.g. the production symbol does not exist),
     write the assertion against the intended behaviour anyway — the
     impl pass will make it pass.

3. An additional bullet at the **end of `## Output discipline`**, in
   the **implement-prompt** only:

   - **No deferred work.** Implement the full task in this single
     pass — no `TODO` / `FIXME` markers, no "I'll add this next time"
     comments, no half-applied edits. If you cannot complete a piece
     of "Files to modify", treat it as a Stop condition and report.

4. `commands/task-setup.md` frontmatter `version` bumped `0.4.0 → 0.5.0`.

In `scripts/cmd-task-impl.sh`, each of the three `--message` strings
passed to `run_aider` (lines ~183-184, ~204-206, ~240-241) gets a
trailing `\n\nRespond in English.` so the directive is reinforced even
when the prompt file is stale. `run_aider`'s body is unchanged.
