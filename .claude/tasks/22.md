# Task 22 — Add `project-setup` command: interactive wizard for first-time project initialization

Target: claude

## Goal
Add a `project-setup` command that acts as a single entry point for
initializing a project with the chosko-llm tooling. It runs in two
phases: a conversational gathering phase that collects all choices
upfront, then a silent execution phase that applies them in a defined
order — no questions mid-execution. Existing commands (`context-build`,
`task-setup`) remain independently usable; `project-setup` orchestrates
them for first-time users.

## Acceptance criteria
- `commands/project-setup.md` exists with valid frontmatter (name, version,
  type, description).
- Gathering phase asks, in order:
  1. VCS: auto-detect git vs Plastic SCM; present detected VCS as default.
     Accept override.
  2. Context layer: build it? If yes — fill CLAUDE.md with synthesized
     project info? If yes — allow user to paste docs for Claude to
     synthesize (not paste verbatim). Also: create AGENTS.md?
  3. Task backlog: set it up?
- After gathering, a summary of all planned actions is shown and the user
  must explicitly confirm before any file is written.
- Execution order is strictly:
  1. Create minimal CLAUDE.md skeleton (only if context-build skipped and
     CLAUDE.md does not exist).
  2. Run context-build logic (if requested) — it writes the navigation
     section and commits its own artifacts.
  3. Fill CLAUDE.md with synthesized project info (if requested) — prose
     synthesized from gathered docs and codebase knowledge, not verbatim paste.
  4. Amend CLAUDE.md with VCS override section (always, based on detected VCS).
  5. Create AGENTS.md (if requested) — minimal file pointing agents to CLAUDE.md.
  6. Run task-setup logic (if requested) — it commits its own artifacts.
  7. Commit project-setup's own artifacts: CLAUDE.md changes from steps 3–4
     and AGENTS.md from step 5, using the correct VCS (git or `cm checkin`).
- VCS section injected into CLAUDE.md maps git commands to Plastic equivalents
  when Plastic is detected; omitted entirely when git is detected (no override needed).
- AGENTS.md is minimal: a one-liner telling other agents to consult CLAUDE.md.
- Smoke test at `tests/smoke/project-setup.md` covers the happy path for
  both git and Plastic scenarios.
- `features.md` context file updated to document the new command.

## Decisions
- `project-setup` orchestrates but does not replace `context-build` and
  `task-setup`. Both remain standalone. This avoids duplication and keeps
  each command's scope clean.
- Commit strategy: sub-commands commit their own artifacts independently;
  `project-setup` commits only what it writes directly (CLAUDE.md project-info
  and VCS sections, AGENTS.md) in a single final commit.
- User-pasted documentation is synthesized into CLAUDE.md prose by Claude, not
  inserted verbatim. This keeps CLAUDE.md concise and actionable.
- VCS section is always evaluated but only written for non-git VCS. For Plastic:
  inject the git→cm substitution table. For git: no section needed, commands
  work as authored.
