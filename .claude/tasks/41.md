# Task 41 — Remove the smoke-test suite and update docs that reference it

Target: claude

## Goal
The `tests/smoke/` manual checklists are considered no longer useful. Delete
the entire smoke-test suite and scrub every reference to it from the
documentation layer so the repo is internally consistent — no doc should
tell a contributor to author a smoke test or point to `tests/smoke/` after
this lands. In the same pass, make CLAUDE.md state positively that this
project has no test suite by design, so the absence reads as a deliberate
choice rather than an omission.

## Acceptance criteria
- The `tests/smoke/` directory and all its files (including README.md) are
  deleted.
- `CLAUDE.md` "Testing" section is reworded to (a) keep the
  `Testing policy for /task-implement: skip-tests` line intact, (b) state
  explicitly that this project has no test suite by design, and (c) drop the
  `tests/smoke/*.md` sentence and its link entirely.
- `CLAUDE.md` "add a smoke-test checklist" step in "When asked to add a new
  feature" is removed (remaining steps renumbered).
- `README.md` repo-layout table no longer lists the `tests/smoke/` row.
- `docs/authoring-guide.md` no longer contains the two "Add a smoke-test
  entry at tests/smoke/<name>.md" steps (surrounding steps renumbered).
- `.claude/context/features.md` no longer documents smoke-test files, the
  "Smoke tests bump alongside version" bullet, or the `tests/smoke/README.md`
  cross-refs.
- `.claude/context/INDEX.md` features.md row no longer mentions `tests/smoke/`.
- `.claude/domain/context-workflow.md` no longer lists the
  `tests/smoke/context-*.md` collateral bullet.
- `grep -ri smoke` over CLAUDE.md, README.md, docs/, and .claude/context|domain
  returns nothing (historical `.claude/TASKS.md` / `.claude/tasks/*.md`
  entries are exempt).
- `VERSION` bumped (patch) per the repo's versioning rule.

## Decisions
- The "no test suite by design" statement lives in CLAUDE.md's "Testing"
  section (co-located with the existing `skip-tests` policy line) rather than
  in a new section or README, keeping the testing stance in one place.
- Historical backlog records (`.claude/TASKS.md`, `.claude/tasks/*.md`) are
  left untouched — the task workflow forbids editing past task entries, and
  their `Files:` lines are a record of what those tasks touched at the time.
- `commands/task-setup.md` is left untouched — its "test files / smoke
  checklists" mentions describe scaffolding for an end-user's project
  generically, not this repo's `tests/smoke/`, so they don't break when the
  directory is deleted.
- VERSION bump is a patch: no shipped command/skill behavior changes, only
  this repo's docs and its (non-shipped) test collateral.
