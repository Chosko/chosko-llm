# Task 26 — Add a `--no-commit` flag to commands that auto-commit changes

Target: claude

## Goal
Give each command that currently commits automatically an opt-in `--no-commit`
flag that writes its changes and leaves them uncommitted for review instead of
committing. Default (no flag) behavior is unchanged. Commands in scope:
`/task-add`, `/task-clean`, `/task-implement`, and `/context-update`.

## Acceptance criteria
- Each in-scope command parses `--no-commit` and, when present, performs all
  its file writes but makes no commit, ending with a report that lists what was
  written and reminds the user to commit.
- Without `--no-commit`, each command behaves exactly as before (auto-commit).
- `/task-implement --no-commit`: still flips task statuses in `TASKS.md` and
  runs the full TDD sequence, but skips the per-task commit, leaving each
  task's changes uncommitted.
- `--commit` (task 25) and `--no-commit` are mutually exclusive — passing both
  is an error with a clear message.
- Each touched command's smoke test and its entry in
  `.claude/context/features.md` are updated; each touched command's `version`
  is bumped per the authoring guide.

## Decisions
- Shared flag convention with task 25 (same parsing, wording, and mutual
  exclusivity). See [[task-25]] for the opt-in-commit side.
- context-update is in scope only because task 24 made it auto-commit.
