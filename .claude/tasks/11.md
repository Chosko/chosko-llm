# Task 11 — cmd-update — skip already-up-to-date and locally-ahead features in --all mode

## Description

`chosko-llm update --all` currently re-copies every installed feature that
has a source in the managed clone, even when:

- the installed version equals the source version (nothing changed), or
- the installed version is *higher* than the source version (the local install
  is ahead — overwriting it would be a downgrade).

This task makes `update --all` compare the installed version against the
source version before copying. Only features where the installed version is
**strictly lower** than the source version are re-copied. All other cases
are skipped with a log line:

| Condition                          | Action  | Log line                                                          |
|------------------------------------|---------|-------------------------------------------------------------------|
| installed == source                | skip    | `Already up-to-date: command 'foo' (v0.3.0)`                     |
| installed > source                 | skip    | `Local version ahead: command 'foo' (local v0.4.0, latest v0.3.0) — skipping` |
| installed < source                 | update  | `Updated command 'foo' -> v0.4.0` (existing behavior)            |
| either version is empty/unversioned| skip    | `Skipping command 'foo': version unreadable — update manually`    |

The version comparison must handle standard semver (MAJOR.MINOR.PATCH) using
awk (no external tools beyond awk/sed/grep — see conventions). Do not use
`sort -V`; it is a GNU extension, not POSIX.

This change applies **only to the `--all` path**. The single-feature
`chosko-llm update <feature>` path is unchanged.

Suggested version-comparison helper (pure awk, add near the top of the
`--all` block or as a named function):

```bash
# version_cmp <a> <b>
# Prints -1, 0, or 1 — the ordering of semver strings a vs b.
# Returns non-zero exit code if either string is empty / non-semver.
version_cmp() {
  local a="$1" b="$2"
  [ -n "$a" ] && [ -n "$b" ] || return 1
  awk -v a="$a" -v b="$b" 'BEGIN {
    n = split(a, av, "."); m = split(b, bv, ".")
    if (n != 3 || m != 3) exit 2
    for (i = 1; i <= 3; i++) {
      if (av[i]+0 > bv[i]+0) { print  1; exit 0 }
      if (av[i]+0 < bv[i]+0) { print -1; exit 0 }
    }
    print 0; exit 0
  }'
}
```

Usage in the loop:

```bash
inst_ver="$(read_frontmatter_field "$f" version || true)"
src_ver="$(read_frontmatter_field "$(src_command_path "$base")" version || true)"
cmp="$(version_cmp "$inst_ver" "$src_ver" 2>/dev/null || echo "?")"
case "$cmp" in
  0)  log_info "Already up-to-date: command '$base' (v$inst_ver)"; continue ;;
  1)  log_info "Local version ahead: command '$base' (local v$inst_ver, latest v$src_ver) — skipping"; continue ;;
  -1) ;; # fall through to update_one
  *)  log_warn "Skipping command '$base': version unreadable — update manually"; continue ;;
esac
update_one command "$base"
any=1
```

### Conventions to follow

- Scripts start with `set -euo pipefail` and `source lib.sh`.
- No new external dependencies. Pure bash + awk/sed/grep. **Do not use
  `sort -V`** — it is a GNU extension, not POSIX. Use the awk helper above.
- Use `read_frontmatter_field` from `lib.sh` for all frontmatter reads.
- Use path helpers — never hardcode `~/.chosko-llm` or `~/.claude`.
- Version comparison is MAJOR.MINOR.PATCH semver via awk only.
- If either version is empty or not a valid X.Y.Z string, skip the feature
  with a warning rather than erroring out — `--all` is best-effort.
- The `any` counter must not increment for skipped features; only increment
  when `update_one` is actually called.
- Use `log_info` for up-to-date and local-ahead skips; `log_warn` for
  unreadable-version skips.

### Out of scope

- Do not change the single-feature `chosko-llm update <feature>` path.
- Do not add a `--force` flag to bypass the version check.
- Do not add pre-release / build-metadata semver parsing (X.Y.Z only).
- Do not change the "no source in managed clone" skip-with-warning behavior.
- Do not bump versions of unrelated skills or commands.
- Do not refactor `update_one` or any code outside the `--all` block.

### Behavior change

Before: `update --all` copies every installed feature that has a source,
regardless of version.

After: `update --all` applies a three-way version comparison:
- `installed < latest` → re-copy (update).
- `installed == latest` → skip, log `Already up-to-date`.
- `installed > latest` → skip, log `Local version ahead … — skipping`.
- either version unreadable → skip, log warning.

`Nothing to update.` still prints when `any` remains 0 (all features
skipped or no candidates).
