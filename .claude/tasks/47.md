# Task 47 — Single-source the non-git VCS rule — strengthen the injected ## VCS section, remove the per-command pointers

Target: claude

## Goal
The git→cm (Plastic SCM) mapping is already single-sourced: it lives once as
a template inside commands/project-setup.md, which injects it as a `## VCS`
section into the target project's CLAUDE.md — and that section is
self-instructing ("When any command instructs you to run a git command,
substitute the Plastic equivalent") and loaded in every session. The 3-line
"NON-GIT VCS: if the project's CLAUDE.md carries a `## VCS` mapping…"
pointer paragraph duplicated across 6 commands is therefore redundant.
Strengthen the injected template so it explicitly overrides literal git
instructions in any command or skill, then delete the 6 pointers.

## Acceptance criteria
- The `## VCS` template in commands/project-setup.md explicitly states that
  the mapping OVERRIDES literal git commands in any command, skill, or
  instruction the model follows in this project (staging/commit semantics
  included — Plastic has no staging area).
- The NON-GIT VCS pointer paragraph is removed from all 6 carriers:
  task-setup, task-enrich, context-build, context-update,
  refactor-codebase, refactor-tests.
- project-setup's own PHASE logic that injects/updates the section is
  unchanged apart from the template wording (it already updates an
  existing `## VCS` section in place — that behavior must keep working so
  re-running /project-setup migrates old projects to the strong wording).
- Frontmatter version patch-bumped on all 7 command files; VERSION
  patch-bumped.

## Decisions
- Option chosen over shipping a global claude-md artifact carrying the
  rule: the project CLAUDE.md is always in context, so a global artifact
  would add permanent context weight for git-only users while covering the
  same case.
- Projects that already carry the old injected wording keep working (the
  old text also instructs substitution); re-running /project-setup
  refreshes them to the override wording.
- task-implement carries no VCS pointer, so there is no interplay with
  task 44.

  NON-GIT VCS paragraph)
- .claude/context/features.md (update if it mentions the per-command
  pointers)
