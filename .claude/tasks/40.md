# Task 40 — Make /task-implement explain the manual step before asking the human to confirm it

Target: claude

## Goal
On a `claude+human` (or `human`) task, when /task-implement reaches a
manual-intervention checkpoint it currently jumps straight to asking the
user whether they've done their part — without first telling them clearly
what to do. Fix the checkpoint protocol so Claude ALWAYS first writes a
short explanatory paragraph ("Now I need your manual intervention — please
do the following…") laying out the concrete step, and only THEN asks for
confirmation and waits. Explain-first, ask-second.

## Acceptance criteria
- Step 2 ("Walk the user through") of the checkpoint protocol in
  commands/task-implement.md is reworded to require an explanatory
  lead-in paragraph BEFORE any confirmation question — naming the manual
  action(s) concretely (real paths/names) so the human knows exactly what
  to do without asking back.
- The ordering is explicit: explain the step, THEN ask + wait (current
  Step 3). The confirmation question never appears on its own without the
  preceding explanation.
- The wording covers both targets (claude+human and human) since both use
  the same protocol.
- tests/smoke/task-implement.md is updated so the checkpoint walkthrough
  check verifies Claude explains the step before asking for confirmation.
- Root VERSION is bumped (patch).
