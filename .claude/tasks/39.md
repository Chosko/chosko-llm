# Task 39 — Persist a project's "no automated test suite" testing policy so /task-implement stops asking every run

Target: claude

## Goal
`/task-implement`'s NO-TEST-SUITE MODE asks the A/B question (scaffold a
suite vs. skip-tests mode) every single run when no test suite is
detected. It also has a detection gap: a `tests/` directory alone counts
as "has a test suite" even when it only holds manual checklists (as in
this repo's `tests/smoke/*.md`, which `tests/smoke/README.md` explicitly
states are unautomated). Let a project record its testing-mode choice
once, durably, in `CLAUDE.md`, and have `/task-implement` honor it
without asking — while still preserving skip-tests mode's per-task
"Proceed?" confirmation.

## Acceptance criteria
- `commands/task-implement.md` documents a `Testing policy for
  /task-implement:` marker line that a project's `CLAUDE.md` may contain,
  with value `skip-tests` or `full-tdd`.
- Before running the existing heuristic detection (LOCATING THE TEST
  RUNNER / NO-TEST-SUITE MODE), `/task-implement` reads `CLAUDE.md` (if
  present) and looks for that marker line:
  - `skip-tests` → enter skip-tests mode directly, no A/B prompt. Still
    show a brief one-line note referencing the declared policy, and still
    run the existing per-task "Proceed?" confirmation before each task —
    skip-tests mode's behavior is otherwise unchanged.
  - `full-tdd` → treat the project as having a test suite (skip
    NO-TEST-SUITE MODE entirely) even if the heuristic alone wouldn't
    detect one, and proceed to LOCATING THE TEST RUNNER for the actual
    command to run.
  - No marker found → fall back to the existing heuristic detection and
    the existing A/B prompt, unchanged from today.
- This repo's own `CLAUDE.md` gets the marker: `Testing policy for
  /task-implement: skip-tests`, with a one- or two-line note pointing at
  `tests/smoke/README.md` for why (manual checklists, no automation).
- `tests/smoke/task-implement.md` gets a scenario verifying that with the
  marker present, a dirty-tree-free run of `/task-implement <N>` on a
  project with no detectable automated suite skips straight to skip-tests
  mode without asking A/B, and the per-task "Proceed?" still fires.

## Decisions
- The marker lives in `CLAUDE.md` rather than a new dedicated state file,
  because this repo's own `CLAUDE.md` already states "no lockfiles or
  state files" as a design rule for the shipped CLI, and `CLAUDE.md` is
  human-authored documentation, not generated state — consistent with how
  the rest of the project's conventions (test command, code style) are
  already recorded there.
- `full-tdd` is included symmetrically with `skip-tests`, even though the
  immediate ask was only for the skip-tests case, because a marker that
  can only assert one of the two outcomes would be a confusing half
  feature — a project with an unconventional or hard-to-detect test setup
  should be able to force full-TDD mode the same way.
