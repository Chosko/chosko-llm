# Task 28 — Add a `show` suggestion to `chosko-llm ls` output

Target: claude

## Goal
Extend the actionable-suggestions block printed at the end of
`chosko-llm ls` so it also reminds the user they can inspect any feature
with `chosko-llm show <feature>`. Today the block (in `scripts/cmd-ls.sh`,
the TTY-gated section near the end) only suggests `add` for installable
features, `update` for updatable ones, or prints "Everything is up to
date." The `show` hint applies to any listed feature, so it should always
be available when suggestions are shown.

## Acceptance criteria
- When `ls` prints its suggestions (interactive stdout, at least one row
  displayed), a line suggesting `chosko-llm show <feature>` to inspect a
  feature is printed.
- The `show` hint appears regardless of whether the add/update hints fired,
  and is present on the "everything up to date" path too (so the inspect
  hint is never lost just because there is nothing to add/update).
- Piped or redirected output (`[ -t 1 ]` false) stays a clean table — no
  suggestion lines, including the new one.
- `tests/smoke/cmd-ls.md` gains a check asserting the `show` suggestion
  appears in interactive output and is absent when stdout is redirected.

## Decisions
- Place the `show` hint last in the suggestions block, after any add/update
  hints. It is unconditional (it applies to any feature), so it does not
  depend on the `suggested` flag the other hints use.
- No version bump: `cmd-ls.sh` is a CLI script, not a versioned
  command/skill feature, so there is no frontmatter `version` to bump.

## Hints
- scripts/cmd-ls.sh — the suggestions block at the end of `list_all`
  (the `if [ "$found" -eq 1 ] && [ -t 1 ]` section); add the `show` line
  and adjust the "Everything is up to date." branch so the hint still shows.
- tests/smoke/cmd-ls.md — add the assertion(s).
