# Task 18 — Print actionable suggestions at the end of `ls` output

Target: claude

## Goal
After the `ls` table, print a short block of actionable next-step suggestions
derived from the rows actually shown — e.g. how to install an available
feature, or update outdated ones. The table itself is unchanged; suggestions
are additive and only appear on an interactive terminal so piped/scripted use
of `ls` is unaffected.

## Acceptance criteria
- After the table, `cmd-ls` prints a suggestions block computed from the rows
  that were displayed (respecting the active filter).
- Suggestions are written to **stdout but only when stdout is a TTY**
  (`[ -t 1 ]`); when `ls` is piped or redirected, no suggestion lines are
  emitted and the table bytes are unchanged.
- The block is separated from the table by one blank line and is only printed
  when there is at least one suggestion to make.
- Installable features (status `not installed`):
  - exactly one  → `Run 'chosko-llm add <name>' to install it.`
  - two or more  → `Run 'chosko-llm add <feature>' to install one, or 'chosko-llm add --all' to install all N.`
- Updatable features (status `updatable`):
  - exactly one  → `Run 'chosko-llm update <name>' to update it.`
  - two or more  → `Run 'chosko-llm update --all' to update all N updatable features.`
- Filter-aware: under `--installed`, `not installed` rows are hidden, so the
  install suggestion is not shown; under `--available`, the update suggestion
  is only shown for rows that actually carry an installed version.
- When rows exist but nothing is installable or updatable, print a single
  `Everything is up to date.` line (still TTY-gated).
- Counts in the messages reflect the filtered, displayed rows — not the whole
  catalog.
- The existing `No features found.` path (empty result) is unchanged.

## Decisions
- TTY-gated stdout (per user): suggestions show on an interactive terminal and
  are suppressed when piped, so scripts parsing `ls` see only the table.
- Single-vs-plural thresholds: one match names the specific feature; two or
  more point at the `--all` variant.
- Suggestions are derived from the same status values the table already
  computes (`not installed` / `updatable` / `up-to-date` / `local only`) — no
  new status logic. Accumulate the relevant names/counts while iterating the
  three kinds, then emit the block once at the end.
- `local only` features get no action suggestion (nothing to add or update);
  they are simply excluded from the counts.
- Optional, only if task 17 (`show`) has landed: the single-updatable hint may
  also suggest `chosko-llm show <name> --diff` to preview the change. Not a
  precondition — core suggestions stand alone.
