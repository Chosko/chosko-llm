# Task 43 — Add "mirrored copy" sync markers to the duplicated test-runner tables

Target: claude

## Goal
The test-runner inference heuristics (pytest/npm/cargo/go decision tree) are
maintained as two near-verbatim hand-synced copies: LOCATING THE TEST RUNNER
in commands/task-implement.md and TEST RUNNER INFERENCE in
commands/task-setup.md. True single-sourcing is blocked by the install model
(commands ship as single self-contained copied files), so as an interim
measure add an explicit sync marker at the top of each section so drift is
flagged at edit time.

## Acceptance criteria
- Both sections open with a clearly visible marker naming the counterpart
  file AND section, e.g.
  "MIRRORED COPY — this table is duplicated in commands/task-setup.md
  (TEST RUNNER INFERENCE). Any edit here must be mirrored there."
- The heuristics themselves are not reworded — marker lines only.
- Both commands' frontmatter `version:` patch-bumped; VERSION patch-bumped.

## Decisions
- Markers instead of extraction: commands are copy-installed single files
  executed in arbitrary target projects, so a reference to a shared fragment
  would dangle at the point of use. Real single-sourcing becomes possible
  after the skill conversion (task 44) or an authoring-time build step —
  both out of scope here.
