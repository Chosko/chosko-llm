# Task 33 — Make `uninstall` invokable via the chosko-llm proxy

Target: claude

## Goal
Uninstalling currently requires locating and running `./uninstall.sh` from
a repo checkout. Every other operation is reachable through the
`chosko-llm` proxy, but `uninstall` is not in the proxy's subcommand
dispatch. Add `chosko-llm uninstall` so teardown is a first-class CLI verb,
with an explicit confirmation gate so it can't be triggered accidentally,
and update the docs to lead with it.

## Acceptance criteria
- `chosko-llm uninstall` runs the existing uninstall flow (proxy removal,
  optional feature removal, optional managed-clone removal).
- Before performing ANY destructive action, the flow asks for an explicit
  top-level confirmation and aborts on anything other than yes. Today the
  proxy removal (step 1 of uninstall.sh) runs unconditionally — that gap is
  closed so nothing is deleted without a confirmation.
- The per-step prompts for feature removal and managed-clone removal are
  preserved.
- A new `scripts/cmd-uninstall.sh` wrapper is the proxy's dispatch target,
  consistent with the other `cmd-*.sh` subcommands; it execs the root
  `uninstall.sh` (the single source of teardown logic — not duplicated).
- `bin/chosko-llm` adds `uninstall` to its subcommand `case`.
- Running `chosko-llm uninstall` still works even though it deletes the
  managed clone it is executing from (exec the script first; the running
  process keeps its file handle).
- `chosko-llm help` / `docs/cli-help.txt` list `uninstall`.
- README's "Uninstall" section documents `chosko-llm uninstall` as the
  primary path, with `./uninstall.sh` noted as the standalone fallback.

## Decisions
- Thin wrapper (`scripts/cmd-uninstall.sh` → root `uninstall.sh`) over
  special-casing the proxy, confirmed with the user: keeps the proxy's
  uniform `cmd-<sub>.sh` dispatch and a single teardown implementation.
- An explicit top-level confirmation is added because exposing uninstall as
  a CLI verb makes accidental invocation easier than running a script by
  path; the existing per-step prompts are kept on top of it.
- `uninstall.sh` stays at the repo root and remains runnable standalone; the
  install instructions that reference `./uninstall.sh` keep working.

## Hints
- scripts/cmd-uninstall.sh — new wrapper; `exec "$CHOSKO_LLM_HOME/uninstall.sh"
  "$@"`. Other cmd-*.sh start with `set -euo pipefail` and source lib.sh, but
  uninstall.sh is self-contained and does NOT source lib.sh — keep the wrapper
  minimal.
- uninstall.sh — add the top-level confirmation gate before step 1 (the
  `confirm()` helper already exists in the script). Honor a non-interactive
  escape hatch if one is warranted (e.g. `--yes`/`-y` or a `FORCE` env), but
  default to prompting.
- bin/chosko-llm — add `uninstall` to the `case` near line 22-24.
- docs/cli-help.txt — add the `uninstall` line.
- README.md — "Uninstall" section (~line 82).
- .claude/context/cli-entry.md — proxy dispatch is documented here; update.
- tests/smoke/cmd-uninstall.md — new smoke checklist.
- NOTE: this changes bin/chosko-llm, so pre-existing proxies won't gain the
  verb until users re-run install.sh — call this out in the commit message.
