# Task 45 — Add a parity guard between the /task-implement prompt and cmd-task-impl.sh

Target: claude

## Goal
The 8-step task workflow is encoded twice — once as the /task-implement
prompt (English) and once as scripts/cmd-task-impl.sh (bash) — and nothing
asserts they stay behaviourally aligned; drift is currently caught by a
human, if at all. Add a mechanical guard that compares the cheap-to-check
invariants across the two: the status-tag vocabulary ([MISSING], [STUBBED],
[INCORRECT], [PARTIAL], [IN PROGRESS], [DONE], [SKIP]) and the step count
of the per-task flow. It won't prove full parity, but it turns silent drift
into a caught diff.

## Acceptance criteria
- A new scripts/check-task-parity.sh (set -euo pipefail, sources lib.sh,
  bash+grep/awk only — no new dependencies) exits 0 today and exits
  non-zero with a clear message if a status tag or workflow step exists in
  one artifact but not the other.
- It reads the prompt from its post-task-44 location
  (skills/task-implement/SKILL.md and its supporting files) and the bash
  side from scripts/cmd-task-impl.sh / scripts/lib-task-external.sh.
- Sanity-checked both ways: temporarily adding a fake status tag to either
  side makes it fail.
- Documented in docs/authoring-guide.md: run it whenever either artifact
  changes.
- VERSION patch-bumped.

## Decisions
- A standalone check script rather than a smoke-checklist item: task 41 is
  slated to remove the smoke-test suite, and a script is mechanical rather
  than relying on a human to eyeball two files.
