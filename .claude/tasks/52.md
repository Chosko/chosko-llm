# Task 52 ‚Äî Add the unity-mcp-skill skill as a shipped feature

Target: claude

## Goal
Fetch the `unity-mcp-skill` skill currently installed globally at
`~/.claude/skills/unity-mcp-skill/` and vendor it into this repo as a
shipped skill feature under `skills/unity-mcp-skill/`, so it installs and
updates through the `chosko-llm` CLI like any other feature. The skill is
a Unity-MCP operator guide (resource-first workflow, tool categories,
batch/screenshot/test patterns) with two reference files. It complements
the existing Unity story (`/unity-mcp-setup`, the Unity-MCP checkpoints in
`/task-implement`) by giving Claude a reusable operator guide for driving
the Unity editor over MCP.

## Acceptance criteria
- `skills/unity-mcp-skill/SKILL.md` exists with the global skill's body
  copied verbatim, plus a complete repo frontmatter block: `name:
  unity-mcp-skill`, `version: 0.1.0`, `type: skill`, and the existing
  `description:`.
- `skills/unity-mcp-skill/references/tools-reference.md` and
  `references/workflows.md` are copied verbatim from the global install.
- The global install's `.unity-mcp-skill-sync` marker is NOT copied.
- `./bin/chosko-llm ls --available` (in a clone where install.sh has run)
  lists `unity-mcp-skill` at version 0.1.0.
- `.claude/context/features.md` documents the new skill in the "Currently
  shipped" list.
- README.md mentions the skill where the Unity features are described.
- Root VERSION is bumped (minor ‚Äî new feature).

## Decisions
- Frontmatter reconciliation is required: the global SKILL.md has `name:
  unity-mcp-orchestrator` and no `version`/`type`. The repo hard-rule
  forces `name` = folder name and requires `version`, so the vendored copy
  uses `name: unity-mcp-skill`, `version: 0.1.0` (new-feature start per the
  authoring guide), `type: skill`. Body and description are otherwise
  verbatim.
- Exclude the `.unity-mcp-skill-sync` marker ‚Äî it's an
  install-management artifact of the global install, not skill content.
- Ship the reference files verbatim even though the SKILL body links two
  files that don't exist in the source (`resources-reference.md`,
  `references/probuilder-guide.md`). This is a faithful fetch; fixing
  upstream dangling links is out of scope.
- VERSION bump is minor (new shipped feature), not patch.
