# Task 27 — Audit and update user- and developer-facing docs to reflect recent changes

Target: claude

## Goal
After the project-setup command (task 22) and the commit-control work (tasks
23-26) land, sweep the user- and developer-facing text surfaces and bring
anything stale into line with actual behavior — the README, the CLI help text,
the authoring guide, and each command's `# Usage:` header and frontmatter
`description`. The navigation context/domain layer is explicitly OUT of scope:
it is maintained separately via `/context-update`.

## Acceptance criteria
- `README.md` reflects the project-setup command and the `--commit` /
  `--no-commit` conventions wherever it lists or describes commands.
- `docs/cli-help.txt` is consistent with current CLI behavior.
- `docs/authoring-guide.md` is accurate against the current feature set.
- Every command's `# Usage:` header and frontmatter `description` matches its
  actual current behavior and flags (verify; fix any that drifted).
- `tests/smoke/README.md` is accurate.
- Each discrepancy is fixed in place; a surface already correct is noted as
  verified rather than edited.
- Docs only — no command behavior changes.

## Decisions
- Documentation-only sweep; it must not alter any command's behavior.
- The `.claude/context/` and `.claude/domain/` navigation layer is excluded —
  the user refreshes it with `/context-update` separately.
- Gated on 23-26 so it audits their final landed state (22 is already done).
