## Goal
Some projects (e.g. Unity) have implementation steps that cannot be executed
by an agent at all â€” the user must act in an external tool (a game-engine

## Decisions
- "Test commands" in the request was read as the task-* commands â€” the
  description is about `/task-implement`'s implementation phase and
  `/task-add` authoring.
- Human involvement is expressed through the existing `Target:` field
  (`claude+human`, `human`) rather than a new TASKS.md field â€” user's
  choice; Target already mirrors between index and body, so backlog views
  get the flag for free. Token normalized to `claude+human` (no spaces)
  for parse-friendliness.
- `## Manual interventions` is a proper `##` section (schema-consistent)
  whose first line is the âš  warning; the mechanism is generic, Unity is
  only the example.
- Verification is independent: Claude checks outcomes itself instead of
  trusting the user's confirmation, per the motivating failure case
  ("user says the prefab exists, but the file is missing").
- For `Target: human`, Claude still owns status flips and the commit â€”
  only production changes are the user's.
