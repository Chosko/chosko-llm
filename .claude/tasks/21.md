# Task 21 — Improve colours: color-code feature kinds and implement colors for `show` command

Target: claude

## Goal

Two related colour improvements. First, in `cmd-ls.sh` the KIND column
(`command`, `skill`, `claude-md`) is always printed with `$C_DIM`; each kind
should instead get a distinct, consistent colour so the three types are visually
distinguishable at a glance. Second, `cmd-show.sh` currently prints all output
in plain text; its metadata block (`Kind`, `Status`, `Installed`, `Latest`, the
header line, and the footer tip) should receive the same semantic colours that
`cmd-ls.sh` already uses — kind colours for the Kind field, status colours for
the Status field, dim for `—` placeholders.

## Acceptance criteria

- `chosko-llm ls`: each KIND value (`command`, `skill`, `claude-md`) renders in
  its own colour, not `C_DIM`. The three kinds are visually distinct.
- `chosko-llm show <feature>`: the header line uses `C_BOLD`, the `Kind:` value
  uses the same colour as the corresponding kind in `ls`, the `Status:` value
  uses the same semantic colour as `ls` (`C_GREEN` / `C_YELLOW` / `C_DIM` /
  `C_CYAN`), `—` placeholders in `Installed:` / `Latest:` are `C_DIM`.
- All colour is gated on `_use_color_stdout` (no ANSI when piped, redirected, or
  `NO_COLOR` set). The metadata block always prints; only colour is conditional.
- `tests/smoke/colors.md` and `tests/smoke/cmd-show.md` are updated to document
  the new expected colours.

## Decisions

- If the three kind colours cannot be sourced from the existing palette
  (`C_GREEN`, `C_YELLOW`, `C_CYAN`, `C_DIM`, `C_BOLD`, `C_RESET`) without
  semantic collision (all three status-bearing colours are already claimed), add
  one or two new colour variables (e.g. `C_MAGENTA`, `C_BLUE`) to `lib.sh` and
  update the palette guidance in `.claude/context/shared-lib.md`. The deciding
  criterion is readability: two kinds sharing a colour is worse than a slight
  palette extension.
- Kind colours should be consistent between `ls` and `show` — use the same
  mapping in both scripts, even if that means extracting a helper or duplicating
  a short `case` block.
