# Task 7 — Refactoring — add `/refactor-tests` command

## Description
Add a new command `/refactor-tests` that finds test files exceeding a
configurable line threshold (default: 300 lines) and splits them into
smaller, focused files — nothing else. No renaming, no deduplication, no
import sorting, no constants extraction. The sole concern is the
`splitting` phase from the `refactor-codebase` model, applied only to
test files.

The command must run the test suite before splitting to establish a green
baseline, and again after each split to confirm no test broke. If the
baseline is red, it stops and reports. If a post-split run is red, it
stops and reports — it does not continue to the next file.

The command does NOT need a full plan-approval gate like
`/refactor-codebase`. It should propose the list of files it will split
(with estimated line counts) and ask once for confirmation, then execute.

### Behavior change
After this task lands:
- `/refactor-tests` is a new available command.
- When invoked it: (1) runs the test suite; if red, stops. (2) scans
  for test files over the threshold; reports the list with line counts
  and asks for confirmation. (3) For each confirmed file, splits it and
  re-runs the suite; stops if any split produces a red suite.
- Tests in the original file are relocated to the split files without
  alteration — signatures, fixture references, and imports are preserved.

### Tests
Add `tests/smoke/refactor-tests.md` following the format in
`tests/smoke/README.md`. Checklist should cover:
