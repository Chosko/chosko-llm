# Task 17 — Add `show` command to inspect a single feature

Target: claude

## Goal
Add a `chosko-llm show <feature>` subcommand that inspects one feature in
depth: its name, kind, installed version, latest version, status, description,
and path. It reuses the status vocabulary and the path/frontmatter helpers
that `cmd-ls` already relies on, but focuses on a single feature and can also
display the feature's body and a diff. After the main output it prints
actionable next-step suggestions with the exact commands to run.

## Acceptance criteria
- `bin/chosko-llm` routes `show` → `scripts/cmd-show.sh` (forwarding `$@`);
  unknown subcommands still exit 2.
- `chosko-llm show <feature>` with no flags prints a metadata block: name,
  kind (`command` | `skill` | `claude-md`), installed version, latest version,
  status (`up-to-date` | `updatable` | `not installed` | `local only` — same
  vocabulary as `ls`), description, and path. Path is the installed path under
  `$CLAUDE_HOME` when installed, otherwise where it *would* be installed; for
  `claude-md`, the `$CLAUDE_HOME/CLAUDE.md` section anchor.
- A header line states which version is shown: defaults to the **installed**
  copy, falling back to **latest** when the feature is not installed.
- `--installed` shows the installed copy's metadata; if not installed, prints a
  clear "not installed" message (and suggests `chosko-llm add <name>`).
- `--latest` shows the latest (managed-clone) copy's metadata; if the feature
  is local-only (absent from the clone), prints a clear message that there is
  no managed/latest copy.
- `--diff` compares latest vs installed. Without `--content`: a summary
  (the two versions + status). With `--content`: a unified `diff -u` of the two
  bodies. If either copy is missing, prints a clear message + a suggested
  command rather than failing opaquely.
- `--content` is additive: in addition to metadata, it prints the body of the
  selected view (installed / latest, or the diff when `--diff`). Without
  `--content`, no body is printed.
- The three selection flags `--installed` / `--latest` / `--diff` are mutually
  exclusive; combining them is an error (`die`).
- After the main output, a footer prints actionable next steps with concrete
  commands, derived from status: `updatable` → suggest
  `chosko-llm show <name> --diff` and `chosko-llm update <name>`;
  `not installed` → `chosko-llm add <name>`; `up-to-date` → note it is current;
  `local only` → note there is no managed copy. When content was not shown,
  the footer also hints at `--content`.
- Handles all three kinds; resolves bare names and `command:` / `skill:` /
  `claude-md:` prefixes; ambiguous bare names error the same way
  `resolve_feature` does.
- `-h` / `--help` prints usage and exits 0; any unknown flag → `die`.
- `docs/cli-help.txt` documents `show` and its flags.
- `tests/smoke/cmd-show.md` added (model on `tests/smoke/cmd-ls.md`).

## Decisions
- Default view prefers the **installed** copy, falling back to latest — so
  `show` reflects what is active on this machine. (Per user; reverses an
  earlier "latest is canonical" idea.)
- `--content` is a modifier, not a standalone view: the selection flag chooses
  which copy, `--content` adds that copy's body. `--diff --content` yields the
  actual textual diff.
- `--diff` uses `diff -u` (POSIX) — no new dependency (CLAUDE.md hard rule).
- Reuse `cmd-ls`/`lib.sh` machinery: `read_frontmatter_field`, the
  `src_*`/`inst_*` path helpers, `src_claudemd_path`,
  `claudemd_installed_version`, and the existing status logic. The status
  computation may be extracted into a small `lib.sh` helper if it keeps
  `cmd-show.sh` clean, but that is optional.
- `claude-md` specifics: its "path" is the `CLAUDE.md` section anchor, its
  installed body is the delimited section, and `--latest` / `--diff` compare
  against `claude-md/<name>.md` in the managed clone.
- Bare-name resolution must consider a feature that is installed but
  local-only (absent from the clone), since `resolve_feature` only inspects
  the managed clone — fall back to `installed_kind` for those.
