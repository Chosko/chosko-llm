# Task 46 — Remove the redundant TOOL DISCIPLINE blocks — the global tool-usage-policy already covers them

Target: claude

## Goal
claude-md/tool-usage-policy.md is injected into the global CLAUDE.md, which
is loaded in every session — so the near-identical TOOL DISCIPLINE blocks in
7 command files restate a policy that is already globally in force. They cost
tokens on every invocation and are a drift surface. Remove the generic
blocks entirely; preserve only genuinely command-specific tool constraints.

## Acceptance criteria
- No command file contains a generic TOOL DISCIPLINE block restating the
  global tool-usage-policy (Read over cat, Edit/Write over redirection,
  right-shell-for-the-tool).
- Command-specific constraints embedded in some blocks (e.g. task-add's
  "Bash/PowerShell are used ONLY by PHASE 5") are NOT lost — each is
  relocated as a one-liner into the section it governs.
- docs/authoring-guide.md states that tool discipline is provided globally
  by claude-md:tool-usage-policy and must not be restated per command, and
  records the soft dependency: installing tool-usage-policy is the
  recommended baseline for all commands.
- Frontmatter version patch-bumped on every command touched; VERSION
  patch-bumped.

## Decisions
- Rely on the global claude-md artifact instead of canonicalizing the
  wording in the authoring guide (the earlier plan) — one source that ships
  into context beats seven synced copies.
- The --commit/--no-commit paragraphs stay per-command: they are each
  command's argument contract, not shared boilerplate, and their semantics
  legitimately vary (authoring vs. maintenance defaults).
- Interplay with task 44: whichever lands second handles task-implement's
  block at its then-current location (command file or SKILL.md).
