# Task 36 — Add a git-log substitute to the Plastic `## VCS` snippet so context-update incremental mode works

Target: claude

## Goal
The `## VCS` section that `/project-setup` injects into a non-git project's
CLAUDE.md (Step 3 of commands/project-setup.md) maps only the commit-side
git commands — `git add`, `git commit`, `git status`, `git rev-parse`,
`git diff`. It has no equivalent for the `git log --after="<date>"
--name-only --pretty=format:` command that `/context-update` MODE A
(smart/incremental update) uses to detect which source files changed since
INDEX.md's "Last updated" date. Because no mapped equivalent exists, Claude
can't substitute one and warns that incremental mode won't work on Plastic
("the date detection step would need a cm-based equivalent"), forcing users
to `full` or `files=`. Add a `cm`-based line to the snippet so a Plastic
project can run `/context-update` in incremental mode with no warning.

## Acceptance criteria
- The `## VCS` snippet in commands/project-setup.md (Step 3) gains a mapping
  entry whose git side is the MODE A file-detection log command
  (`git log --after="<date>" --name-only --pretty=format:`) and whose
  Plastic side is a `cm`-based command that lists the source-file paths
  changed since a given date, dedupable to a flat path list.
- After a fresh `/project-setup` on a Plastic project, running
  `/context-update` in default (MODE A) mode substitutes the mapped `cm`
  command for the git log and proceeds without emitting the
  "incremental mode won't work out of the box" caveat.
- The commit-side mappings and surrounding prose of the snippet are
  unchanged except for the added line.
- commands/project-setup.md's `version:` frontmatter is bumped, and the
  root VERSION file is bumped per semver.

## Decisions
- Scope is the snippet only (user decision). No edit to
  commands/context-update.md: the `## VCS` section's general "when any
  command instructs you to run a git command, substitute the Plastic
  equivalent" rule already covers MODE A once the log command has a mapped
  equivalent — the missing equivalent was the whole cause of the warning.
- The exact Plastic invocation must be verified during implementation.
  A plausible starting point is `cm log --from="date:<date>"` with an
  item/path format (e.g. `--itemformat` or `--xml`) piped through
  `sort -u`, but `cm log`/`cm find` flags vary by Plastic version — confirm
  the syntax that actually yields a flat list of changed item paths before
  writing it into the shipped snippet.
