# Task 9 — task-impl CLI flags: --model / --retries / --map-tokens with env-var fallback

## Description

`chosko-llm task-impl` currently exposes its tunables only through env
vars: `CHOSKO_TASK_IMPL_MODEL` (aider `--model`) and
`CHOSKO_TASK_IMPL_RETRIES` (per-pass retry budget). It does not expose
aider's `--map-tokens` flag at all, so users cannot adjust the repo-map
budget without editing the script.

This task adds three CLI flags to `chosko-llm task-impl` and one new env
var:

1. `--model <name>` — overrides `CHOSKO_TASK_IMPL_MODEL`. Forwarded to
   aider as `--model <name>`. Default unchanged
   (`ollama/qwen2.5-coder:14b`).
2. `--retries <N>` — overrides `CHOSKO_TASK_IMPL_RETRIES`. Used as the
   per-pass retry budget in `implement_one`. Default unchanged (`3`).
3. `--map-tokens <N>` — new knob, with matching env var
   `CHOSKO_TASK_IMPL_AIDER_MAP_TOKENS`. Forwarded to aider as
   `--map-tokens <N>`. **If neither the flag nor the env var is set,
   `--map-tokens` is not passed to aider at all** (aider falls back to
   its own default). The flag value takes precedence over the env var.

Precedence rule (uniform across all three): explicit CLI flag >
environment variable > built-in default. For `--map-tokens` the
"built-in default" is "do not pass the flag to aider".

Both `--flag value` and `--flag=value` forms are accepted, and flags
may be interleaved with positional task numbers (e.g.
`chosko-llm task-impl 3 --retries 2 4`).

### Behavior change

After this task lands:

1. `chosko-llm task-impl` accepts three new flags:
   - `--model <name>` / `--model=<name>`
   - `--retries <N>` / `--retries=<N>`
   - `--map-tokens <N>` / `--map-tokens=<N>`
   Each may appear at any position relative to the positional task numbers (e.g. before, between, or after). `--` is **not** required as a separator.

2. Precedence: CLI flag value > env var value > built-in default.
   - `--model`: env var `CHOSKO_TASK_IMPL_MODEL`, default `ollama/qwen2.5-coder:14b`.
   - `--retries`: env var `CHOSKO_TASK_IMPL_RETRIES`, default `3`.
   - `--map-tokens`: env var `CHOSKO_TASK_IMPL_AIDER_MAP_TOKENS`, default unset (flag is omitted from the aider invocation entirely).

3. The aider invocation in `run_aider` includes `--map-tokens <N>` if and only if `AIDER_MAP_TOKENS` is non-empty after argument parsing. When omitted, aider's own default applies.

4. Bad input is rejected with `die` before any aider invocation:
   - `--retries` or `--map-tokens` whose value is not `^[0-9]+$`,
   - any of the three flags with no value (end of args / next token starts with `--`),
   - an unknown flag (anything starting with `--` other than the three above).

5. `usage()` lists the three new flags under an `Options:` section and adds `CHOSKO_TASK_IMPL_AIDER_MAP_TOKENS` under `Environment:`.
