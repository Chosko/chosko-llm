# Task 50 — Offer to run /unity-mcp-setup from /project-setup on Unity projects

Target: claude

## Goal
On Unity projects, /project-setup should offer to run /unity-mcp-setup as
part of first-time init, so MCP wiring is discoverable without the user
knowing the command exists. /project-setup only offers and delegates — it
contains none of the MCP setup logic itself.

## Acceptance criteria
- In the GATHER phase (near the existing Unity subsection 1f, which already
  probes ProjectVersion.txt), when the project is Unity, /project-setup asks
  whether to run /unity-mcp-setup; the choice is surfaced in the single
  confirmation summary.
- In the EXECUTE phase, if accepted, it invokes /unity-mcp-setup (passing
  through --commit semantics consistent with the other delegated
  sub-commands); if declined or non-Unity, it does nothing here.
- No MCP detection/registration logic is duplicated into project-setup — it
  strictly delegates to the task-48 command.
- features.md project-setup entry updated to mention the offer; root VERSION
  bumped (minor).

## Decisions
- project-setup only offers + delegates; all MCP logic stays in
  /unity-mcp-setup so the two don't drift.

  sub-command invocation).

- VERSION.
