# Task 23 — Remove the commit-scaffolding prompt from `/task-setup`; leave changes uncommitted

Target: claude

## Goal
`/task-setup` currently ends with an optional "commit scaffolding? [y/N]"
phase. Make it a pure authoring command instead: it writes the backlog
scaffolding and leaves everything uncommitted in the working tree, matching
`/context-build` and the other authoring commands. The user reviews and
commits when ready.

## Acceptance criteria
- The `PHASE — OFFER COMMIT` section is removed from `commands/task-setup.md`,
  along with the `WRITTEN`-list commit machinery, the commit-related entries
  in TOOL DISCIPLINE, and the commit-related DO NOT bullets.
- task-setup still tracks what it wrote and ends with a report of created vs.
  already-present artifacts, plus an explicit reminder that nothing was
  committed and the user should commit when ready.
- task-setup no longer runs any git/VCS command — the only remaining shell
  use is `mkdir`/`chmod` for scaffolding.
- `commands/project-setup.md` no longer references declining task-setup's
  commit prompt: the upfront notice, Step 5 (task backlog), and the DO NOT
  list are updated to reflect that task-setup now leaves output uncommitted
  by default.
- `tests/smoke/task-setup.md` and `tests/smoke/project-setup.md` drop the
  commit-prompt expectations.
- The task-setup entry in `.claude/context/features.md` no longer says it
  "optionally commits" / "offers to commit".
- task-setup's `version` is bumped per the authoring guide.

## Decisions
- task-setup joins the uncommitted-by-default authoring group. The `--commit`
  opt-in is added separately in task 25, not here — this task only removes the
  prompt and the commit behavior.
