# Task 15 — task-add — thin body by default, --enrich option

Target: claude

## Goal

Update the `task-add` skill so it produces thin task bodies by default (Goal,
Acceptance criteria, Decisions, Hints — no snippets, no required-reading lists,
no conventions blocks). Add an `--enrich` option that creates an enriched body
in a single authoring pass by reading the `task-enrich` skill for format
guidance, rather than duplicating enrichment logic inside task-add.

## Acceptance criteria

- Default `task-add` (no flag) produces a thin body matching the schema defined
  in task 13: Goal, Acceptance criteria, Decisions (when applicable), Hints
  (always present).
- `task-add --enrich` produces an enriched body directly, in one shot. It does
  not write a thin body first and then call task-enrich as a second step.
- When `--enrich` is used, task-add reads `commands/task-enrich.md` to
  understand the enriched format and applies that format during authoring. No
  enrichment logic is duplicated in task-add itself.
- `Target: claude` is written for default runs; `Target: local` for `--enrich`
  runs.
- The `Target:` field is also written into the TASKS.md summary block.
- The conversational flow (task title, goal discussion, decision capture) is
  preserved for both modes.
- Version bumped (minor — new observable behaviour via `--enrich`).
- Smoke test updated to cover both default and `--enrich` paths.

## Decisions

- One-shot enrichment in task-add (not two-step thin→enrich) avoids a
  transient thin body that would immediately be replaced and keeps the user's
  conversation context intact throughout.
- task-add reads task-enrich's SKILL.md for the enriched format definition
  rather than hardcoding it. This makes task-enrich the single source of truth
  for what "enriched" means.
- The `--enrich` flag name is preferred over `--for-local` or `--local` because
  it describes the action (enriching) not the destination (local LLM), keeping
  the option meaningful even if the target audience changes.

## Hints

- commands/task-add.md — primary file to update
- commands/task-enrich.md — read for enriched format guidance (must exist,
  task 14 precondition)
- tests/smoke/task-add.md — update to cover both modes
