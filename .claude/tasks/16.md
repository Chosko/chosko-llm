# Task 16 — task-implement — warn on target: local tasks

Target: claude

## Goal

Update `task-implement` to read the `Target:` field from task body files and
emit a one-line informational warning when implementing a `target: local` task.
The warning makes the mismatch visible without blocking — Claude is always a
valid fallback implementer regardless of how the task was authored.

## Acceptance criteria

- `task-implement` reads the `Target:` field from the body file when present.
- If `Target: local`, a warning is shown before implementation begins:
  "Note: this task was written for a local LLM (target: local) — implementing
  with Claude anyway."
- Implementation proceeds normally after the warning; no confirmation prompt is
  shown.
- If `Target: claude`, `Target:` is absent, or the body file does not exist,
  behaviour is unchanged (no warning, no prompt).
- Version bumped (patch — no new user-facing feature, just a diagnostic hint).
- Smoke test updated with a case for `target: local`.

## Decisions

- Warn but proceed: blocking would strand tasks whose local-LLM run failed and
  need a Claude fallback. The point of the warning is visibility, not a gate.
- No confirmation prompt: the warning is informational. Asking "are you sure?"
  adds friction without value — the user invoked task-implement intentionally.
- Patch bump, not minor: the warning is a diagnostic quality-of-life change,
  not a new capability.
