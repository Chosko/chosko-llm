# Task 44 — Convert /task-implement from a command into a skill with on-demand supporting files

Target: claude

## Goal
commands/task-implement.md is a 600-line monolith (~7k tokens) injected in
full on every invocation, even on the trivial happy path — the
human-in-the-loop protocol, the no-test-suite mode, and the legacy body
schemas are all resident even when none apply. Commands cannot ship
supporting files (single-file copy install), but skills can: cmd-add installs
a skill via recursive copy, so supporting files ride along. Convert
task-implement into skills/task-implement/ where SKILL.md carries the
common path and the conditional sections move to supporting files that the
model reads only when the branch actually applies. Expected common-path
token reduction: ~20–30%.

## Acceptance criteria
- skills/task-implement/SKILL.md exists with full frontmatter (name
  task-implement matching the folder, type, description, version bumped
  minor from the command's current version).
- At minimum the human-in-the-loop protocol, the no-test-suite mode, and
  the legacy body-schema notes live in supporting files under
  skills/task-implement/, and SKILL.md tells the model exactly when to read
  each ("if Target: claude+human, read ./human-in-loop.md").
- SKILL.md's common path (clean tree, tests exist, numbered claude task) is
  self-sufficient without reading any supporting file.
- commands/task-implement.md is removed; no workflow content is lost — every
  branch remains reachable via the supporting files. The sync markers from
  task 43 ride along with the moved test-runner table.
- Installed-user migration is documented (commit message and README note):
  `chosko-llm rm command:task-implement` then
  `chosko-llm add skill:task-implement` (or `chosko-llm update --all`
  behavior verified for the kind change).
- .claude/context/features.md and any cross-references (README,
  docs/authoring-guide.md examples, other commands that name
  commands/task-implement.md) are updated to the new path.
- VERSION minor-bumped.

## Decisions
- This is the repo's first skill — no CLI changes are needed (cmd-add /
  cmd-update / cmd-rm already handle skill:<name>), but verify how
  cmd-update --all treats a feature whose kind changed from command to
  skill; if it can't handle it, document the manual migration in README.
- The user-facing invocation stays /task-implement — only the packaging
  changes.

## Hints
- commands/task-implement.md (source content; delete after the move)
- skills/task-implement/SKILL.md + supporting .md files (new)
- scripts/cmd-add.sh, scripts/cmd-update.sh, scripts/cmd-rm.sh (read-only:
  verify kind-change behavior; no edits expected)
- docs/authoring-guide.md, README.md, .claude/context/features.md
- .claude/domain/task-workflow.md (references the command file)
- tests/smoke/task-implement.md (only if the smoke suite still exists —
  task 41 may have removed it)
- VERSION
