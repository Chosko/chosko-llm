# Task 42 — Fix the double test run in cmd-task-impl.sh — capture once, branch on exit status

Target: claude

## Goal
Steps 5 and 6 of scripts/cmd-task-impl.sh run each test wrapper TWICE per
attempt: once to capture the failure log (`last_log="$(... || true)"`) and
immediately again just to read the exit code (lines ~293–294 for
run-affected-tests.sh, ~312–313 for run-full-tests.sh). On a slow suite this
doubles the wall-clock cost of the most expensive step, on every retry. It is
also a correctness edge: with a flaky suite the two runs can disagree, so the
log handed to aider may not match the verdict that gated the loop. Run each
wrapper once per attempt, capturing output and exit status from the same run.

## Acceptance criteria
- Each test wrapper is invoked exactly once per attempt, in both the
  affected-tests loop and the full-suite loop.
- The failure log passed to aider always comes from the same run that
  produced the pass/fail verdict.
- Works under `set -euo pipefail` (a failing command substitution must not
  abort the script — capture the status explicitly).
- No behavior change otherwise: retry counts, log messages, and die paths
  are unchanged.
- VERSION patch-bumped.

## Decisions
- Capture-once via an explicit status variable (e.g.
  `if last_log="$(cmd 2>&1)"; then ok=1; else ok=0; fi`) rather than tee or
  temp files — keeps the no-new-dependencies rule and the existing style.
