# Task 24 — Make `/context-update` auto-commit its changes

Target: claude

## Goal
`/context-update` currently rewrites context files and leaves them
uncommitted. Add a commit phase at the end that stages exactly the context
files the run modified (including the `INDEX.md` "Last updated" bump) and
commits them, moving context-update into the auto-committing group alongside
`/task-add` and `/task-clean`.

## Acceptance criteria
- `commands/context-update.md` gains a final commit phase that stages only the
  explicit context-layer paths it modified and commits them with a descriptive
  message.
- Staging never uses `git add -A` / `git add .` / `git add -u` or any
  catch-all — only the paths the run actually wrote.
- The existing `git=` change-detection reads (`git diff` / `git log` for
  determining which files to update) are unchanged.
- If the run modified nothing, no commit is made (no empty commit).
- For a non-git VCS, the commit honors the `CLAUDE.md` `## VCS` mapping if
  present (git→cm).
- `tests/smoke/context-update.md`, the context-update entry in
  `.claude/context/features.md`, and `.claude/domain/context-workflow.md` are
  updated to describe the new auto-commit behavior.
- context-update's `version` is bumped per the authoring guide.

## Decisions
- Stage only the context files the run touched — explicit-path discipline
  matching `/task-add`.
- The `--no-commit` escape hatch is added separately in task 26, not here.
